# المنافذ والدومين - شرح مبسّط

## البورتات (المنافذ) التي يعمل عليها التطبيق

| البورت | الخدمة | الوصول |
|--------|--------|--------|
| **3000** | التطبيق الرئيسي (الإدارة) | `http://عنوان-السيرفر:3000` |
| **3001** | تطبيق السائقين | `http://عنوان-السيرفر:3001` |
| **3002** | تطبيق الموظفين | `http://عنوان-السيرفر:3002` |
| **80** | Nginx / Certbot (HTTP للتحقق من الشهادات) | عند وجود دومين فقط |
| **443** | Nginx (HTTPS) | عند وجود دومين فقط |

---

## بدون دومين (استخدام عنوان IP)

إذا كان لديك عنوان IP للسيرفر فقط (مثل `192.168.1.100` أو IP عام):

- **HTTPS مع شهادة Let's Encrypt**: لا يعمل — يحتاج دومين.
- **HTTP**: يعمل عبر البورتات 3000، 3001، 3002.

### طريقة التشغيل بدون دومين

شغّل التطبيقات فقط (Nginx لا يبدأ تلقائياً):

```bash
docker-compose up -d
```

الوصول:
- الرئيسي: `http://192.168.1.100:3000`
- السائقين: `http://192.168.1.100:3001`
- الموظفين: `http://192.168.1.100:3002`

## من أين أحصل على دومين؟

### 1. دومين مجاني (فرعي)

خدمات تعطيك دومين فرعي مجاني يشير إلى IP سيرفرك:

| الخدمة | الموقع | مثال |
|--------|--------|------|
| **DuckDNS** | duckdns.org | delivery.duckdns.org |
| **No-IP** | noip.com | delivery.ddns.net |
| **FreeDNS** | freedns.afraid.org | delivery.mooo.com |

الخطوات العامة:
1. سجّل في الموقع.
2. أنشئ دومين فرعي وربطه بعنوان IP السيرفر.
3. ضع هذا الدومين في ملف `.env` كـ `DOMAIN=delivery.duckdns.org`.

### 2. دومين مدفوع

شراء دومين من مزوّد (مثل Namecheap، GoDaddy، شركات عربية). الأسعار غالباً من حوالي 10–15 دولار سنوياً.

### 3. على نفس الشبكة المحلية (بدون إنترنت)

استخدم عنوان IP للسيرفر على الشبكة المحلية، مثلاً:
- `http://192.168.1.50:3000`

## البورت 80 محجوز لتطبيق آخر

إذا كان البورت 80 مستخدماً (مثلاً Nginx أو تطبيق آخر):

استخدم بورتات بديلة وشهادة ذاتية (HTTPS يعمل لكن المتصفح قد يعرض تحذيراً):

```bash
docker-compose -f docker-compose.yml -f docker-compose.override-ports.yml --profile https up -d
```

الوصول عبر HTTPS على البورت **8443**:
- الرئيسي: `https://IP-السيرفر:8443`
- السائقين: `https://IP-السيرفر:8443/driver`
- الموظفين: `https://IP-السيرفر:8443/employee`

**ملاحظة**: Let's Encrypt لا يعمل لأنه يحتاج البورت 80. الشهادة هنا ذاتية (لتشفير الاتصال فقط).

---

## إضافة التطبيق للـ reverse proxy الموجود (مثل Nginx)

إذا كان عندك Nginx آخر على 80/443، أضف المسارات التالية لتطبيق التوصيل:

```nginx
location / {
    proxy_pass http://127.0.0.1:3000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-Proto $scheme;
}
location /driver {
    proxy_pass http://127.0.0.1:3001;
    # ... نفس الهيدرز
}
location /employee {
    proxy_pass http://127.0.0.1:3002;
    # ...
}
```

## ملخص سريع

| الحالة | البورتات | HTTPS؟ |
|--------|----------|--------|
| بدون دومين | 3000, 3001, 3002 | لا (HTTP فقط) |
| مع دومين + البورت 80 متاح | 80, 443 | نعم (Let's Encrypt) |
| البورت 80 محجوز | 8443 | نعم (شهادة ذاتية) |
