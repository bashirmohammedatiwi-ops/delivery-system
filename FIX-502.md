# حل خطأ 502 على /employee/ و /driver/

## الأوامر السريعة (نفّذ على السيرفر)

```bash
cd /opt/delivery-system

# 1. سحب آخر التحديثات
git pull

# 2. إعادة بناء وتشغيل كل الخدمات (بما فيها Nginx + HTTPS)
docker compose --profile https up -d --build

# 3. انتظر 30 ثانية ثم تحقق
docker ps
```

يجب أن ترى 4 حاويات تعمل: `delivery-system` و `delivery-employee-web` و `delivery-driver-web` و `delivery-nginx`.

## إذا استمر الخطأ 502

```bash
# عرض سجلات Nginx
docker logs delivery-nginx --tail 50

# عرض سجلات تطبيق الموظفين
docker logs delivery-employee-web --tail 30

# إعادة تشغيل الخدمات
docker compose --profile https restart
```

## ملاحظة

- تطبيق الموظفين (`/employee/`) والسائق (`/driver/`) يعتمدان على السيرفر الرئيسي (البورت 3000).
- إذا كان السيرفر الرئيسي لم يكتمل تشغيله بعد، قد يظهر 502 لفترة وجيزة — انتظر دقيقة ثم حدّث الصفحة.
